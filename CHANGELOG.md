# Changelog

## [0.0.6] – 2026-05-02

`RealtimeClient` now connects through the backend's IoT custom JWT
authorizer. Pre-0.0.6 the client expected a SigV4-signed presigned
URL and mqtt.js's default `mqtt` WS subprotocol; the backend has
moved to an unsigned custom-authorizer flow that needs the JWT in
the MQTT CONNECT username and `mqttv5.0` as the WS subprotocol —
without the SDK update every connect attempt is rejected with
`AUTHORIZATION_FAILURE`.

### Added
- `IotCredentials.authorizerName` and `IotCredentials.authToken`
  re-exported from `/auth/iot-credentials`.

### Fixed
- MQTT CONNECT now passes `username = credentials.authToken` (the
  bearer JWT) and `wsOptions.protocol = "mqttv5.0"`. Subscribes
  start succeeding against AWS IoT Core's unsigned custom authorizer.
- `protocolVersion` bumped to `5` to match the WS subprotocol.

## [0.0.5] – 2026-05-02

The 0.0.5 release rebuilds the transport layer on top of an
OpenAPI-generated client and standardises the public surface around
string UUIDs. Every integration was audited for re-entrancy and
shutdown-safety alongside the rewrite.

### Breaking

- **All IDs are `string` UUIDs on the public boundary.** `orgId`,
  `appId`, `channelId`, `environmentId`, `eventId`, `traceId`,
  `endpointId`, `userId`, `apiKeyId`, `releaseId`, `agentId`, `spanId`,
  and `parentEventId` no longer accept `number`. Integrations still
  accept `number` for `channelId` as a v0.0.4 softening — a one-time
  `console.warn` is emitted via `coerceChannelId(...)` and the value is
  stringified.
- **HTTP layer rewritten on top of `@hey-api/openapi-ts`.** The
  generated client lives at `src/_internal/api/` and is regenerated by
  `bun run codegen` against a running backend on `:3000`. SDK code goes
  through `client.invoke(op, args)` — a single chokepoint that owns
  retries, fail-open, and exception mapping. The hand-rolled
  `openapi-fetch` wrappers under `src/transport.ts` and the per-resource
  fetch helpers were removed.
- **`WebSocketClient` and `connectWebSocket()` aliases removed.** Use
  `RealtimeClient` and `connectRealtime()`.
- **`ChannelsResource.subscribe()` SSE shim removed.** Subscribe in
  realtime via `connectRealtime()`.
- **`events.list()` returns `EventListResponseDto`**
  (`{ data, meta }`), not a bare array. Read `.data` for the events.
- **Models moved to flat `axonpush.models` re-exports of generated
  types.** `App`, `Channel`, `Event` (formerly `AxonEvent`),
  `EventDetails`, `EventType`, `Environment`, `WebhookEndpoint`,
  `WebhookDelivery`, `ApiKey`, `Organization`, `User`, `TraceListItem`,
  `TraceSummary` are now type-only re-exports of their `*ResponseDto`
  generated forms. Import them from `@axonpush/sdk`.
- **Constructor argument shape stabilised.** `new AxonPush(options?)`;
  every option falls through to its `AXONPUSH_*` env var when omitted.
  The legacy positional-argument form is gone.

### Added

- **`ChannelsResource.list(appId)`** — returns the channels under an app.
  Backed by a new `GET /channel?appId=` route on the backend.
- **`EventType` widens to accept arbitrary strings while keeping
  autocomplete** for the canonical members. Defined as
  `CanonicalEventType | (string & {})`. The closed enum is exported as
  `CanonicalEventType` for callers who want to enforce it.
- **`WebhookDeliveryStatus`** exported from the public surface so
  callers can pattern-match on `delivery.status` directly.
- **`AxonPush.connectRealtime` is now strongly typed** as
  `(opts?: RealtimeOptions) => Promise<RealtimeClient>` — no more
  `Promise<unknown>` cast at every call site.
- **`Event.id` is populated alongside `Event.eventId`.** The backend's
  `IdAliasInterceptor` aliases the primary key to `id` on every
  response; `EventIngestResponseDto` now declares both, so callers can
  rely on either field without a cast.
- **Settings-style env-var resolution.** `AXONPUSH_API_KEY`,
  `AXONPUSH_TENANT_ID`, `AXONPUSH_ORG_ID`, `AXONPUSH_APP_ID`,
  `AXONPUSH_BASE_URL`, `AXONPUSH_ENVIRONMENT`, `AXONPUSH_IOT_ENDPOINT`,
  `AXONPUSH_WS_URL`, `AXONPUSH_TIMEOUT`, `AXONPUSH_MAX_RETRIES`,
  `AXONPUSH_FAIL_OPEN`. `new AxonPush()` with zero args now works.
- **Backend exception envelope** (`{ code, message, hint, requestId }`)
  is parsed onto every `AxonPushError`. Subclasses preserve
  `statusCode`. `RateLimitError.retryAfter` is parsed from the
  `Retry-After` header.
- **`RetryableError` tagging mixin.** `APIConnectionError`,
  `RateLimitError`, and `ServerError` are recognised as retryable; user
  code can subclass `RetryableError` to opt extra errors in.
- **Built-in retry loop** with exponential backoff
  `[250, 500, 1000, 2000, 4000] ms` honouring `Retry-After`. Configurable
  via `maxRetries`.
- **Examples directory.** Ten runnable examples in `examples/` covering
  quickstart, tracing, realtime, multi-agent fan-out, webhooks, error
  handling, LangChain, winston, Sentry, and OpenTelemetry.

### Improved

- **`BackgroundPublisher`:** bounded queue with explicit overflow policy
  (`drop-oldest` | `drop-newest` | `block`), graceful flush on `close`,
  per-event error isolation, and serverless detection that warns when
  callers should call `flush()` (or wrap with `flushAfterInvocation`).
- **Re-entrancy guards on `pino` / `winston` / `console` integrations**
  via `AsyncLocalStorage`, so SDK-internal logging cannot loop through
  user-installed handlers.
- **`langchain` / `langgraph` / `deepagents` / `otel` propagate
  `parentRunId` → `parentEventId`** instead of flattening every span
  under a single trace root.
- **`anthropic.ts`** records `usage.input_tokens`,
  `usage.output_tokens`, and `stop_reason`, plus a `streamMessage` async
  generator for token-by-token capture.
- **`console.ts`** restores stdio on `process.on('exit' | 'beforeExit'
  | 'uncaughtException')`, so a crash mid-capture doesn't leave the
  process with redirected `console.*`.
- **Realtime credential refresh** uses backoff `[5, 15, 30, 60] s` (was
  a hardcoded 30 s). Refresh is only scheduled after the broker confirms
  the connection.
- **Topic builders** sanitise non-alphanumeric segments (e.g.
  `agent.error` → `agent_error`) so AWS IoT Core accepts them.

### Migration guide (0.0.4 → 0.0.5)

1. **Replace numeric IDs with string UUIDs** in every call to
   `apps.*`, `channels.*`, `webhooks.*`, `events.publish({ channelId })`,
   etc. Compile errors will point at every site.
2. **Replace `connectWebSocket()` with `connectRealtime()`** and rename
   imports of `WebSocketClient` to `RealtimeClient`. The public methods
   (`connect`, `subscribe`, `unsubscribe`, `publish`, `onEvent`,
   `disconnect`, `wait`) are unchanged.
3. **Update `channels.subscribe(...)` callers** to subscribe through
   `realtime.subscribe({ channelId }, callback)` instead.
4. **Adjust `events.list(channelId)` callers** to read `.data` from the
   returned `EventListResponseDto`. Pagination cursors are surfaced on
   `.meta`.
5. **Drop hand-imported model paths.** Replace
   `import { ... } from "@axonpush/sdk/dist/schema"` with
   `import { ... } from "@axonpush/sdk"`.
6. **Optional:** delete any custom retry/backoff wrappers — the SDK now
   handles `RetryableError` natively. Keep your wrapper only if you need
   a non-default policy.

## 0.0.4 and earlier

See `git log` and the previous CHANGELOG entries reproduced below.

### 0.1.0 (pre-0.0.5 internal numbering)

- Realtime transport replaced with MQTT-over-WSS against AWS IoT Core.
  `WebSocketClient` aliased to `RealtimeClient`. Presigned WSS URLs
  fetched from `GET /auth/iot-credentials` with 60 s pre-emptive
  refresh. `socket.io-client` removed; `mqtt` (^5) added.
- SSE removed. `SSESubscription` and `client.channels.subscribe(...)` /
  `subscribeToEvent(...)` kept as deprecation shims.
- `events.search()` / `events.list()` Lucene removed. Both calls now
  accept `EventQueryParams` (`channelId?`, `appId?`, `environmentId?`,
  `eventType?`, `agentId?`, `traceId?`, `since?`, `until?`,
  `payloadFilter?`, `cursor?`, `limit?`).
- `AxonPushConfig` / `AxonPushOptions` gained optional `wsUrl`,
  `iotEndpoint`, `orgId`, `appId`. `wsUrl` / `iotEndpoint` are
  decoupled from `baseUrl`.

### 0.0.11

- **MQTT topic shape changed.** Topics now embed an environment slug
  between the org and app segments
  (`axonpush/{orgId}/{envSlug}/{appId}/{channelId}/{eventType}/{agentId}`).
- **Environments are org-level.** `client.environments` exposes `list`,
  `create`, `update`, `delete`, `promoteToDefault`. The previous
  per-app endpoint `GET /apps/:appId/environments` was removed.
- **Schema regenerated.** Several response types renamed to their
  `*ResponseDto` forms.
- **ID parameters are now strings** on `apps.*`, `channels.*`,
  `apiKeys.revoke`, `webhooks.*`. `events.list(channelId)` /
  `events.publish({ channelId })` accepted `string | number`.
